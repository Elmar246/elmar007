git команды

1. Rebase

Команда git rebase - используется для объединения двух веток.

как работаеть git rebase - последовательно берет все коммиты из выбранной ветки и заново применяет их к новой ветке.

2. Fetch

команда git fetch - агружает коммиты, файлы и ссылки из удаленного репозитория в ваш локальный репозиторий.

как работаеть fetch git - сливает все данные с проекта, которые находятся в нашем remote репозитории

3. Merge

командаgit merge - собрать воедино разветвленную историю и объединения двух веток

зачем нужен merge commit - содержит в себе все изменения целевой ветки мерджа, которые отсутствуют в текущей

  • Main - ветка на которой ваш проект
  • Develop - ветка разработки
  • Feature - ветка для разработки новых функций, которые далее сливаются с develop
  • Release - ветка, которая создается, при завершении проектаи и с этого момента новые функции добавить больше нельзя
